Introduction

Python is a dynamically semantic, interpreted, object-oriented, high-level programming language created in the late 1980’s by Guido van Rossum and first released in 1991. Van Rossum was a fan of the BBC comedy series “Monty Python’s Flying Circus” from the 1970’s, having even read published scripts from the show, so when he needed a name for his new language he chose the name Python. It was short, unique, and even a little mysterious making it a perfect fit for what Van Rossum was looking for in a name. Python’s claim to fame, however, is due more to it’s overall usefulness and functionality than the origin of it’s name. Programmers love Python because it’s easy to learn with a syntax that emphasizes readability, and since there is no compilation step the edit-test-debug cycle is extremely fast.

Python is attractive for Rapid Application Development and for use as a scripting language used to ‘glue’ existing components together thanks to it’s high-level built in data structures combined with dynamic typing and dynamic binding. It supports modules and packages for program modularity and code reuse and you can access the Python interpreter and the extensive standard library free of charge.

In this tutorial we will be covering some basic Python codes while we create a simple program that will print a picture in the terminal and then expand the program to print a different picture based on a number entered by the user. Hopefully you find this to be a fun and informative way to begin your journey as a Python developer.

Encode The Pictures

First we're going to start by encoding the pictures. To do this we will use what is called a list. Lists are used to store multiple items in a single variable. Lists are only one of 4 built-in data types in Python used to store collections of data. The other three are Tuple, Set, and Dictionary each one having different qualities and usage, but in this example we will be focusing on lists.

List items are indexed, the first list item having an index of [0] and the second list item having an index of [1] and so on. They are also orderd, changeable, and allow duplicate values. Ordered means that the items on the list have a defined order that will not change and if you add new items to a list the items will be placed at the end of the list. Changeable means tht we can change, add, and remove items in a list after the list has been created. Also, since the lists are indexed, the lists can have items with the same value or “duplicate values”.

List items can be of any data type and can also contain different data types in the same list. They are created using the name of the list followed by an equal sign and then square brackets containing the list items with each item being separated by a comma with strings being inside quotation marks. A list can also contain more than one set of list items, or square brackets, separated by a comma, which is the type of list we'll be using to encode our pictures.

For this tutorial I have 5 images already encoded, but if you would like to add more, simply draw out the picture you want to create on a piece of graph paper using the characters and/or special characters you would like to use. In this tutorial we will be using /, |, \, _, X, *, and spaces to encode our pictures. You can add additional characters if you would like just remember to update the code in your function to decode and print your pictures, as well as to add any new pictures you create yourself to your picture selector. Once you have drawn your picture out on graph paper, use the following key to encode your picture placing each row on the graph paper within it's own set of brackets and each cell in that row representing a single pixel in your picture.

  • 0 for empty space
  • 1 for /
  • 2 for \
  • 3 for |
  • 4 for _
  • 5 for X
  • 6 for *

Create A Picture Converter Function

A function is a block of code that must be defined and called. You can pass parameters into a function or tie functions together. A function can also return data as a result. A function is defined using the def keyword followed by the name of the function like this:

def pic_convert(picture):

Notice that the name of the function has an underscore in place of a space, this is because in Python funtion names cannot contain spaces otherwise it will cause an error during execution of the program.

Now we have to tell the interpreter what this function is supposed to do when it’s called. We want our program to carry out the same function on each pixel in each row, so that's the first thing we have to tell our function to do. Note I'm the previously discussed line of code in the following examples to show the tab structure which indicates a hiearchy in python.

			
def pic_convert(picture):
      for row in picture:
            for pixel in row:

Now we can tell our function what to do for each pixel in our picture, to do this we will be using an elif statement. Simply put it means that if the pixel is represented by this number, print this symbol in it's place, else if the pixel is represented by this other number print this symbol, else, or otherwise if it's not any of those numbers, print this symbol. For our function the code would look like this:

      for row in picture:
            for pixel in row:

                        print('/', end='')

                        print("_", end='')

                        print('X', end='')
                  elif (pixel == 6):
                        print('*', end='')
                  else:
                        print(' ', end='')
            print('')

Now let's break that down a little bit. The tab structure indicates the heirarchy order of the function. It simply tells the program that for each row in the picture and each pixel in the row, if the value of that pixel is equal to 1 print “/” and if not go to next part of the elif statement and the next until if finds a matching value or has reached the final else statement and has printed a space or character for each pixel in the row. The final print('') statement is lined up with the line of code that says “for each pixel in row” which essentially loops the function back to start the next row and so on and so forth until all rows have been printed. Notice in the elif statements there are 2 equal signs instead of just one. This is because python uses equal signs when assigning names to lists, dictionaries, ect. Therefore when the equal sign is being used to compare a literal value and not assign a name or value then you must use 2 equal signs in a row to signify this to the python interpreter.

Now, in the final else statement it says “print(' ', end='')” , there is a space between the apostrophes in the beginning of the statement that indicates a blank pixel or a space in the picture, and then after the comma there's the code “.., end='')” (with no space between the apostrophes), which tells the program that once it has printed the pixel to print the following pixel on the same line and not on a new line. Without this line of code after each of our pixel options each pixel would be printed on it's own line and our picture wouldn't look like a picture.
